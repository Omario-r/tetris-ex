# Документация прототипа: «Explosive Template Tetris» (MVP)

## 0. Краткое общее описание

Это мобильная головоломка в духе тетриса: сверху падают фигуры, их можно двигать и вращать, а цель уровня — собрать на поле заданную “деталь” по фиксированному шаблону. Главная особенность — у каждой падающей фигуры есть шанс один раз во время падения стать взрывной и помочь “вырезать” лишние блоки.

В обычном режиме фигура падает и упирается в уже стоящие блоки, как в тетрисе. Если сделать фигуру взрывной, она перестаёт сталкиваться с препятствиями и проходит сквозь любые блоки до самого низа, при этом её можно взорвать в любой момент. Взрыв удаляет только те клетки, которые находятся прямо под формой фигуры в момент взрыва, пустоты после взрыва остаются на месте.

Уровень считается пройденным, когда все клетки шаблона заполнены, и вокруг контура детали нет соприкасающихся “мусорных” блоков даже по диагонали (8‑соседство), при этом касание стен и дна разрешено.

## 1. Цель и скоуп

Прототип проверяет механику формирования фиксированной «детали» по шаблону уровня: игрок складывает блоки как в тетрисе, но может перевести текущую падающую фигуру в взрывной режим и взорвать её, чтобы точечно удалить мешающие клетки.

В прототипе важно:
- Управление (влево/вправо/поворот/ускорение падения).
- Взрывной режим: понятность, полезность, частота применения.
- Понимание игроком: «почему деталь ещё не готова» (индикатор контактов по 8‑соседству).

Вне скоупа текущего прототипа: большая картина (мета-уровень поверх объектов), генерация уровней, магазин, мультиплеер.

***

## 2. Core loop

- Уровень задаёт фиксированный шаблон детали (Target).
- Спавнится падающая фигура в обычном режиме.
- Игрок двигает/вращает фигуру как в тетрисе.
- Игрок может перевести текущую фигуру в «взрывной режим» — **ровно один раз для этой фигуры**, обратно нельзя.
- В взрывном режиме фигура падает сквозь установленные блоки до дна; игрок может взорвать её в любой момент.
- Взрыв удаляет только клетки поля, которые совпадают с отпечатком фигуры в момент взрыва.
- **Сразу после взрыва** (и после каждой фиксации обычной фигуры) проверяется условие победы; если выполнено — уровень завершается, новая фигура не спавнится.

## 3. Правила и определения (MVP)

### 3.1 Поле
Прямоугольная сетка. Стены и дно — границы поля.

**Дефолт прототипа (MVP):** поле 10×20.

### 3.2 Набор фигур
Стандартные 7 тетромино (I, O, T, S, Z, L, J).

**Рандомизация (MVP):** Random Bag (7-bag): в каждом «мешке» ровно по одной фигуре каждого типа, порядок внутри мешка случайный.

### 3.3 Target (шаблон уровня)
Уровень задаёт множество клеток S — они должны быть заполнены блоками, чтобы деталь считалась собранной. «Дыр» внутри детали нет: важна только заполненность клеток S.

**Дефолт прототипа (MVP):** Target — квадрат 6×6, приклеен ко дну поля. Его левая граница начинается с колонки 2 (0-based), то есть Target занимает колонки 2..7.
внутри окна 6×6 целевая фигура — крест толщиной 2 клетки

### 3.4 Halo (контурная зона)
Ореол H — все клетки поля, прилегающие к любой клетке S по стороне или диагонали (8‑соседство), но не входящие в S.

Требование: все клетки H внутри поля должны быть пустыми. Если часть H выходит за границы поля — она игнорируется, поэтому Target может касаться стен и дна.

### 3.5 «Мусор»
Любые занятые клетки, которые не являются частью S. Если мусорный блок оказывается в зоне H — это нарушение условия победы.

### 3.6 Режимы падающей фигуры
Падающая фигура имеет два режима:
- **Normal:** стандартный тетрис — коллизии с границами и установленными блоками; при невозможности падать — фиксация.
- **Explosive:** коллизии только с границами поля; установленные блоки игнорируются, фигура проходит сквозь них; при касании дна — авто-взрыв.

Перевод Normal → Explosive разрешён **ровно один раз для каждой новой падающей фигуры**. Обратно изменить нельзя.

### 3.7 Взрыв
Взрыв можно активировать в любой момент, пока фигура в режиме Explosive. Взрыв удаляет только те клетки поля, которые находятся под отпечатком фигуры в момент взрыва. Все остальные блоки не затрагиваются.

### 3.8 Гравитация после взрыва
Отсутствует. Пустоты после взрыва остаются на месте.
Игрок расчищает их последующими взрывами.

### 3.9 Победа и поражение (MVP)
- **Победа:** все клетки S заняты и все клетки H внутри поля пусты. Проверяется сразу после каждой фиксации или взрыва.
- **Поражение:** невозможность заспавнить новую фигуру (переполнение) либо исчерпан лимит ходов (если добавим в уровень).

## 4. Архитектура (без Flame)

**Принцип:** игровая логика отделена от UI.

- **Engine (чистый Dart, без Flutter):** правила и состояние игры (поле, фигуры, режимы, взрыв, гравитация, Halo-проверка, условия победы/поражения).
- **Flutter UI:** отображение состояния и ввод игрока (кнопки/жесты), поверх отрисовки поля.
- **Рендеринг поля:** через `CustomPaint/CustomPainter` (рисование на Canvas во время paint-фазы). [api.flutter](https://api.flutter.dev/flutter/rendering/CustomPainter-class.html)
- **Тайминг/loop (если нужен для плавного падения):** через `Ticker`/`AnimationController`, а перерисовка — через `repaint`-механизм `CustomPainter`, чтобы обновлять кадры без лишних build/layout. [stackoverflow](https://stackoverflow.com/questions/62624726/flutter-improve-custompainter-animation-performance)

Зависимости строго однонаправленные: Flutter UI → Engine. Обратно запрещено.

***

## 5. Тестирование (минимальный план)

**Unit-тесты (engine):**
- Halo по 8‑соседству, игнор клеток за границами поля.
- Проверка победы: S заполнена и H пуста.
- Explosive-падение сквозь блоки.
- Взрыв удаляет только отпечаток фигуры.

**Widget-тесты (UI):**
- Подсветка зон S и H.
- Корректная реакция на кнопки/жесты управления.

**Integration-тест (1 сценарий):**
- Сыграть фиксированную последовательность: arm → detonate → получить состояние victory.
- Для воспроизводимости теста генератор фигур должен быть детерминированным: фиксированный seed и/или заранее заданная последовательность выдачи фигур.

Критерий «готово»: все unit/widget тесты зелёные, есть хотя бы один integration-тест, и ручной сценарий уровня проходится.

## 6. Мета-прогрессия

### 6.1 Терминология
- **Уровень** — одна игровая сессия. Игрок собирает один фрагмент.
- **Фрагмент** — форма, которую нужно собрать на уровне.
  После победы встаёт на своё место в объекте.
- **Объект** — визуальная фигура из 4 фрагментов.
- **Большая картина** — мета-слой из 4 объектов. Следующая итерация.

### 6.2 Структура прогресса
  Объект × 4
  └── Фрагмент × 4
      └── Уровень

Прогрессия линейная. При провале — повтор уровня.

### 6.3 Объект 1 — Домик
Симметричный домик со ступенчатой крышей, разбитый на 4 фрагмента:
- Крыша левая (ступенчатый треугольник)
- Крыша правая (зеркальный треугольник)
- Стена левая (прямоугольник)
- Стена правая (прямоугольник)

Фрагменты геометрически стыкуются без перекрытий.

### 6.4 Объекты 2–4
Заглушки из простых прямоугольников —
нужны для проверки навигации между объектами.

### 6.5 Экраны между уровнями
- **После уровня:** показывается объект с фрагментами,
  текущий фрагмент встаёт на место. Переход к следующему уровню.
- **После объекта:** показывается собранный объект целиком.
  Переход к следующему объекту.
