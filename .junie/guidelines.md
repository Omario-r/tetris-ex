# Junie Guidelines — Explosive Tetris

## Спецификация проекта
При каждой задаче читай файл Tetris-ex-spec.md в корне проекта.
Он является единственным источником правды для правил игры, координат,
форматов данных и API. При любом противоречии между промптом и спецификацией
— приоритет у спецификации; сообщи об этом явно перед выполнением.

## Архитектура
- game_engine/ — чистый Dart, НИКАКИХ импортов flutter/* или package:flutter/*.
- Зависимости только вниз: presentation → application → game_engine.
- Обратные зависимости запрещены.

## Обязательные проверки после каждой задачи
- flutter test — все тесты зелёные.
- flutter analyze — нет ошибок и предупреждений.

## Стиль кода
- Dart lints (flutter_lints).
- Имена файлов: snake_case. Классы: PascalCase.
- Тесты лежат в test/ с зеркальной структурой lib/.
- Каждый публичный класс и метод имеет краткий doc-комментарий (///).

## Поведение при невалидных командах
- Все публичные команды GameController возвращают void.
- При невозможности выполнить команду — no-op (state не меняется, исключение не бросается).

## Что НЕ делать без явного указания
- Не добавлять зависимости в pubspec.yaml без согласования.
- Не трогать файлы вне scope задачи.
- Не реализовывать wall kicks, CCW-поворот, анимации — это backlog.
- Не добавлять логику в файлы-заглушки этого шага.
